# HIP.markets Brand System

HIP.markets should feel native to Hyperliquid culture while remaining its own original brand. The cat direction is intentional: Hyperliquid has well-known cat motifs across Hypurr/PURR culture, but HIP.markets should not copy Hyperliquid marks, Hypurr NFTs, or any existing mascot.

## Brand Idea

**The risk sentinel for builder-deployed markets.**

HIP.markets is not a playful memecoin interface. It is an underwriting and operator-vault product. The mascot should therefore feel alert, technical, and responsible: a cat watching oracle cadence, vault health, fee flow, and slashing risk.

## Visual Language

- Dark terminal base for market-operator credibility.
- Mint green for healthy execution, capital readiness, and fee flow.
- Blue for data, oracle relayers, and infrastructure.
- Amber for risk, slashing warnings, and unresolved approval steps.
- Small radii, dense panels, tabular numbers, and restrained motion.

## Mascot Usage

The cat mascot is called **HIP Cat** in the demo. It should appear as:

- a risk sentinel in the operator monitor;
- a brand flourish on README and presentation materials;
- a small favicon/mark in the app chrome;
- never as a replacement for risk disclosures.

The mascot should not dominate the product. The product still needs to look like a serious exchange-operator dashboard.

## Voice

HIP.markets copy should be:

- precise;
- skeptical;
- risk-aware;
- commercially realistic;
- clear about which steps are automated, multisig-controlled, or still unverified.

Avoid fixed-yield claims. Always pair APR references with operating costs, reserves, and slashing risk.
